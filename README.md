# 🏭 AI辅助经营诊断与预警系统

> **把运营经验教给AI，让每个电商卖家都有一个7×24小时的运营诊断师**

---

## 📌 项目背景

天猫家具类目运营中，一个运营专员经常要同时盯着几十个商品链接的周报数据。传统做法是人工逐条排查，效率低且容易遗漏。本项目将运营经验转化为**分层诊断规则引擎**，结合LLM实现自动异常检测、根因推测与行动建议输出。

**当前状态：** MVP已跑通 ✅  
**技术栈：** Python + Pandas + DeepSeek API + Markdown报告输出

## 🔑 核心亮点

### 1. 分层诊断规则引擎

传统AI诊断把"异常"当统一概念处理，但电商运营中**同一个指标在不同品级的含义完全不同**：

| 指标 | A级品（核心款） | B级品（潜力款） | 新品（冷启动） | C级品（长尾款） |
|:---|:---:|:---:|:---:|:---:|
| **退款率** | >5%即🔴预警 | >10%🔴预警 | >30%🔴预警 | >10%🔴预警 |
| **花费超标** | 战略性投入可🟢接受 | 放量期可🟢观察 | 冷启动期🟢正常 | >500元即🔴预警 |
| **成交额下滑** | >15%🔴预警 | 放量期可接受 | 数据量不足不判断 | 反向上行异动🟡关注 |

这种**精细化的运营逻辑**是纯技术背景的人写不出来的——它是真实操盘经验的代码化。

### 2. 三级行动建议体系

每条预警自动输出**紧急/重要/优化**三级行动建议，运营人员可以按优先级直接执行：

> 🔴 **紧急**：暂停全站推广渠道的推广，立即联系供应链确认质检报告  
> 🟡 **重要**：排查近期订单，主动联系高退款客户了解原因  
> 🟢 **优化**：检查详情页描述是否与实物一致，准备修改方案

### 3. 🔄 自我反思机制（最核心的差异化亮点）

诊断报告末尾包含**运营策略反思**模块——系统会自动发现规则设计中的漏洞，比如：

- **"异常基准链式污染"**：新品判定依赖B级品均值，但B级品自身异常时会把标准"带偏"，导致新品漏报
- **"C级品向上异动规则缺失"**：C级品成交额暴涨无法被已有规则捕获，可能是异常订单也可能是有潜力的品

这个机制展示的是**元认知能力**——诊断系统不仅能执行规则，还能反思规则本身是否合理。

## 🚀 快速开始

```bash
# 1. 克隆项目
git clone <your-repo-url>

# 2. 安装依赖
pip install pandas requests

# 3. 配置API密钥（可选，不配置则使用本地分析模式）
# Windows PowerShell:
$env:DEEPSEEK_API_KEY="your-api-key-here"
# Mac/Linux:
# export DEEPSEEK_API_KEY="your-api-key-here"

# 4. 准备周报数据（参考 weekly_data_v2.csv 格式）
# 5. 运行诊断
python diagnosis.py

# 6. 查看报告
open diagnosis_report.md
```

## ⚠️ 数据声明

当前版本使用**模拟数据**进行功能验证和规则测试。MVP阶段有意使用模拟数据：
1. 避免真实数据口径不一致导致第一版跑不通
2. 保护数据隐私
3. 规则逻辑验证通过后，接入真实数据源只需替换数据接入层

## 🧠 项目背后的方法论

这个项目的核心不是"调API"，而是**把运营方法论教给AI**：

- **分层管理思维**：不同品级的商品有不同的管理重点，这是天猫运营的基本功
- **异常≠坏事**：C级品成交暴涨可能是异常订单，也可能是潜力品——需要"向上异动"规则
- **自我反思**：好的诊断系统不仅要输出结果，还要反思自己的诊断规则是否合理
